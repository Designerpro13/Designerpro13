## What am I Like

A Student developer who gets unreasonably invested in solving problems that probably have a "just use X" answer - and then builds the alternative anyway. I work across systems programming, network tools, and security: offline CLIs, browser-based attack surface analyzers, local biometric auth subsystems, LAN file transfer engines, concurrent web crawlers. Stack shifts with the problem - Python for rapid tooling, Go when concurrency and networking matter, Rust when the threat model demands it, Node.js for the occasional midnight bot.

Big on AI-assisted development - not the copilot-autocomplete kind, but actual agentic workflows. Kiro is my absolute go-to, GitHub Copilot on VS Code for when I'm deep in the editor, Lovable and Figma's AI when the problem has a UI. Simple, fast, and efficient - the way tooling should be.

Linux-first (Ubuntu, Zorin, Parrot, Kali, Tails - you name the next!) - I prefer an OS that doesn't treat me like a tenant on my own hardware or hide its background telemetry behind a frosted glass GUI.

Currently exploring post-quantum cryptography and decentralized network architectures - because cloud is great, just not when a big-tech company owns the keys. No daemons, no unnecessary API calls, no biometric data leaving the machine. If it needs 300MB of embedded Chromium, it's not a tool - it's a hostage situation.
